# AGENTS.md

本仓库用于整理哈尔滨工程大学课程资料、复习材料、课程设计样例与课程说明。

## 维护目标

- 保持课程目录结构清晰。
- 保持课程说明文件内容可读、客观、简洁。
- 保留仓库中已有的重要项目信息、贡献信息、上传说明和许可说明。
- 优先补充目录说明、资料来源和适用范围，不随意删除原有内容。

## 文件组织

- 顶层目录使用课程名。
- 课程目录内优先沿用已有分类，如 `PPT`、`资料`、`习题`、`历年题`、`实验`。
- 新建课程说明文件使用 `README.md`。
- 若目录中已存在 `readme.md` 或 `README.MD`，则直接修改现有文件，不重复创建新的 `README.md`。
- 同类资料放入同类目录，避免重复堆叠在课程根目录。

## Markdown 约定

- 使用客观描述，不写维护者的主观看法、推荐语或下一步计划。
- 优先说明目录用途、资料类型、资料来源、适用年级或适用课程阶段。
- 保留原有链接、贡献者名单、上传方法和许可信息。
- 在优化现有文档时，以重排结构、补齐说明、统一格式为主，不随意删减已有有效信息。

## 文件命名

- 文件和目录名不包含空格。
- 文件名可包含年份、题型、用途等信息。
- 不新增无明确含义的缩写目录名。

## 大文件

- 仓库未启用 `git-lfs`。
- 单文件不超过 100 MiB。
- 安装包、缓存、编译产物和临时文件不进入版本库。

## 修改原则

- 小步修改，优先保证原内容可追溯。
- 涉及根目录 `README.md` 时，保留原有项目说明框架。
- 对课程资料本体谨慎处理，优先修改说明文档，不直接改动原资料文件。
