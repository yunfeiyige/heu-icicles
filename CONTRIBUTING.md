# 贡献说明

本仓库同时作为课程资料仓库和站点内容源。目录结构、文件命名和 `README.md` 内容会直接影响 GitHub 页面与站点页面的展示。

## 基本要求

- 文件和目录名不要包含空格。
- 单文件不要超过 100 MiB。
- 仓库未启用 `git-lfs`。
- 保持现有目录结构清晰、稳定。
- 同类资料放入同类目录。

## 目录结构

每门课程目录下通常包含以下内容：

```text
课程名/
├── README.md
├── PPT/
├── 资料/
├── 习题/
├── 历年题/
├── 实验/
└── 其他补充材料
```

## 常见分类

- `PPT`：课件、讲义、课堂展示材料
- `资料`：总结、笔记、速查表、补充阅读
- `习题`：练习题、题解、作业答案
- `历年题`：往年试题、回忆题、真题整理
- `实验`：实验文档、模板、代码、报告
- `README.md`：课程说明、资料来源、使用提示、贡献说明

## 文件和目录命名

- 顶层目录使用课程名。
- 新建课程说明文件使用 `README.md`。
- 如果目录中已存在 `readme.md` 或 `README.MD`，直接修改现有文件，不重复创建新的 `README.md`。
- 文件名中写明年份、用途或版本，例如：
  - `2024-期末回忆.pdf`
  - `实验3-参考代码.zip`
  - `第6章-重点整理.md`

## README 文件内容

课程目录下的 README 文件可包含以下内容：

- 课程名称
- 目录说明
- 资料来源
- 适用年级或教师版本
- 考核方式、重点章节、复习说明

示例：

```md
# 课程名

## 目录说明
- `PPT`：课件与讲义
- `资料`：复习资料与补充笔记

## 补充说明
- 资料来源
- 适用范围
```

## 不上传的内容

- 盗版电子书、付费电子书副本
- 盗版、破解版、绿色版付费软件及其安装包
- 编译产物、工程缓存、无关临时文件
- 无法确认版权状态的整本电子书
- 课程或教师主页已公开提供但未授权再分发的材料

## 大文件

大文件可在 `README.md` 中说明用途，并提供外部下载地址。

## GitHub 网页端提交流程

1. Fork 本项目。
2. 进入对应课程目录。
3. 上传文件或新建子目录。
4. 提交修改。
5. 发起 Pull Request。
